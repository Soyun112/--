# AI 어린이 안심 길찾기 인수인계서

작성일: 2026-07-15  
목적: 현재 MVP를 이어받아 발표·개선 작업을 진행하기 위한 상태 정리

## 1. 서비스 개요

출발지와 목적지를 입력하면 Tmap 보행 경로 후보를 만들고, 공공 안전 데이터로 경로별 안전점수를 계산하는 서비스입니다.

- 부모 모드: 경로 후보별 안전점수·거리·안전시설 비교
- 아이 모드: 선택한 경로의 핵심 안전 키워드와 단계별 길 안내
- 지도: Leaflet + OpenStreetMap
- 백엔드: FastAPI

## 2. 실행 방법 (macOS)

```bash
cd backend
source .venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

브라우저에서 `frontend/index.html`을 엽니다.

- API 상태: <http://127.0.0.1:8000/api/health>
- API 문서: <http://127.0.0.1:8000/docs>

테스트:

```bash
cd backend
source .venv/bin/activate
pytest
```

## 3. 현재 구현 완료 항목

### 경로·안전점수

- Tmap 보행 경로 기본 경로와 우회 경로 A/B를 생성
- CCTV, 어린이보호구역, 사고다발지역, 안전지킴이집, 보안등, 단속카메라, 문서 기반 위험 정보를 바탕으로 안전점수 계산
- 안전점수 동점 시 `대로`·`로` 구간이 더 긴 경로를 우선하고, 다시 동점이면 더 짧은 경로를 선택
- 카드 클릭으로 우회 경로를 직접 선택 가능
- 선택한 경로는 지도 강조, 아이 모드 키워드, 아이 길 안내에 함께 반영

### 아이용 길 안내

- Tmap이 반환한 실제 보행 안내 단계(`description`, `turnType`, 구간 거리)를 백엔드 응답에 포함
- 아이 화면에서는 긴 도로명·출구명 문장을 숨기고 아래처럼 단순화해 표시
  - `↑ 직진 · 160m`
  - `↰ 왼쪽으로 가기 · 63m`
  - `↱ 오른쪽으로 가기 · 45m`
  - `⌖ 도착`
- Tmap 호출 실패로 MOCK 경로가 반환되면 상세 보행 안내는 제공되지 않음

### 장소·시나리오

- 등굣길: `개나리SK뷰5차아파트 → 도성초등학교`
- 학교 → 학원길: `도성초등학교 → 필수학학원`
- 야간 학원 하원길: `필수학학원 → 개나리SK뷰5차아파트`
- 위 장소는 `backend/app/services/geocoding.py`의 데모 장소 사전에 등록되어 외부 검색 결과에 흔들리지 않음
- 출발지·목적지 교체 버튼 제공
- 입력 영역의 위도·경도 표시는 제거됨
- 아이 나이 입력 기능은 HTML/JS에서 주석 처리됨

### UI

- 부모 모드 / 아이 모드 토글
- 아이 모드에는 선택한 경로의 안전 키워드와 길 안내만 표시
- 부모용 긴 AI 리포트는 화면에서 제거됨
- 수동 라이트/다크 모드 토글 제공
- 현재 다크 모드는 사용자가 직접 선택하며 시간 기준 자동 전환은 미구현

### 강남구 CCTV

- 사용자 제공 파일 `backend/app/data/cctv_gangnam.csv`를 읽어 지도 데이터로 제공
- 강남구 CCTV 2,672건
- 선택 경로 주변 140m 이내 CCTV만 지도에 표시
- CCTV 마커에는 카메라 대수·설치 목적·주소가 포함됨

## 4. 중요한 제약 및 수정 필요 항목

### 우선순위 높음: CCTV와 안전점수의 불일치

강남구 CCTV CSV는 현재 **지도 표시용**입니다. 안전점수 계산은 기존 어린이보호구역 데이터의 `cctv_count`만 사용합니다.

따라서 지도에는 CCTV가 보여도 점수에는 반영되지 않을 수 있습니다.

개선 방향:

1. `public_data.get_gangnam_cctvs()`를 안전점수 계산에도 전달
2. 경로 근처 CCTV 수와 CCTV 밀도를 계산
3. 기존 `cctv_density` 점수 항목에 실제 강남구 CCTV 값을 반영

### CCTV 근접 판정 정확도

현재 CCTV는 Tmap 경로의 좌표 점과 140m 이내인지 비교합니다. 긴 선분 중간의 CCTV를 놓칠 가능성이 있습니다.

개선 방향:

- 좌표 점 기준이 아니라 경로 선분(polyline)까지의 최단거리로 판정
- 140m는 발표용 임시 기준값이며 정책적으로 재검토 필요

### 나머지 안전시설 데이터

- 강남구 CCTV 외 어린이보호구역·사고다발지역·안전지킴이집·보안등·단속카메라는 샘플 데이터 또는 검증이 덜 된 공공데이터 사용
- 다른 강남 지역을 검색하면 안전지킴이집·보안등 등이 0개로 나올 수 있음
- 실제 서비스화를 위해서는 지역별 데이터를 수집·DB 적재한 뒤 경로 반경 조회 구조로 바꿔야 함

### 아이용 길 안내 검증

현재 안내는 Tmap 실제 안내 단계를 사용하므로, Tmap 응답이 정상인 경우 방향·횡단보도·거리의 근거는 Tmap에 있습니다.

발표 전 다음을 확인해야 합니다.

1. 시나리오 3개에서 `TMAP_PEDESTRIAN_API`가 반환되는지 확인
2. 지도 경로와 아이 카드의 직진·좌/우회전·도착 순서가 일치하는지 확인
3. 예외적인 Tmap 문구가 `직진`으로 과도하게 단순화되지 않는지 확인
4. Tmap 실패 시 MOCK 경로에 길 안내가 없다는 점을 안내할지 결정

### 큰길 우선 규칙

동점 처리에서 `대로`·`로` 도로명으로 큰길 여부를 추정합니다. Tmap은 도로 폭을 직접 주지 않으므로 완전한 도로 폭 판정은 아닙니다.

## 7. 환경변수 및 보안

- 실제 API 키는 루트 `.env`에만 둡니다.
- `.env`는 커밋하면 안 됩니다.
- 필요한 키: `TMAP_APP_KEY`, `UPSTAGE_API_KEY`, `DATA_GO_KR_SERVICE_KEY` 등
- 키가 없거나 Tmap 요청이 실패하면 일부 기능은 MOCK 데이터로 동작합니다.
