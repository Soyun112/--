# 라이브 데모 스크립트

`backend`를 실행하고 `frontend/index.html`을 브라우저로 열어 아래 순서로 시연한다.
API 키가 없어도 MOCK 데이터로 동일하게 시연 가능하다.

## 데모 시나리오 (프론트엔드 드롭다운에서 선택)

화면 좌측 "데모 시나리오" 드롭다운에서 3개 시나리오를 선택할 수 있고, 각각 선택 후
"선택한 시나리오 좌표 채우기"를 누르면 좌표/나이가 자동 입력된다. 세 시나리오 모두
같은 샘플 데이터(`backend/app/data/sample_*.json`, `sample_documents/`)를 서로 다른
조합으로 지나가도록 설계했다.

| 시나리오 | 출발지 → 목적지 | 보여주는 것 |
| --- | --- | --- |
| 🏠→🏫 등교길 | 집(37.4995, 127.0370) → 학교(37.5035, 127.0410) | 직선 경로 vs 큰길 우회 경로의 CCTV/사고다발지역/문서위험 대비 |
| 🌙 야간 학원 하원길 | 학원(37.5021, 127.0400) → 집(37.4995, 127.0390) | 범죄위험지수가 전반적으로 높은 지역에서도 CCTV·보안등 있는 길을 선별 추천 |
| 💯 만점 시나리오 | 유치원(37.4998, 127.0370) → 학교(37.5025, 127.0410) | 안전지킴이집·보안등·CCTV 밀집 구간 → 안전점수 100점 + 스탬프 전체 획득 |

## 시연 순서 (기본: 🏠→🏫 등교길 시나리오)

1. **문제 제기**: "카카오맵으로 이 두 지점을 검색하면 최단거리인 직선/골목 경로를 추천합니다.
   하지만 이 경로에는 교통사고다발지역이 2곳, 그리고 실제 지자체 안전진단 보고서에서
   지적된 무단횡단 위험구간이 있습니다."
2. 화면에서 "🏠→🏫 등교길" 시나리오 선택 → "선택한 시나리오 좌표 채우기" → "안전 경로 찾기" 클릭.
3. 결과 화면에서 경로 후보 카드들을 비교한다(ID는 MOCK 모드에서 `route-direct`/`route-grid-a`/
   `route-grid-b`, LIVE 모드(Tmap 키 있음)에서는 `route-tmap-direct`/`route-tmap-via-a`/
   `route-tmap-via-b`로 다르게 표시되지만 스토리는 동일):
   - **직선(최단거리) 경로**가 가장 낮은 점수(대략 55~58점대)를 받는다 — 사고다발지역 2곳 +
     문서상 위험지적구간 통과 + 안전지킴이집·보안등 없음.
   - **CCTV·보호구역·안전지킴이집·보안등이 많은 우회 경로**가 가장 높은 점수(대략 75점대)로
     추천된다.
   - (참고: MOCK/LIVE 모드에 따라 Tmap이 실제로 반환하는 경로 형태가 달라 점수는 ±수 점 오차가
     있을 수 있으나, 상대적 순위는 항상 동일하게 유지된다 — 상대 정규화 방식 덕분.)
4. **추가 시연(선택)**: "🌙 야간 학원 하원길" 시나리오로 전환해 "범죄위험지수가 높은 지역에서도
   저희 서비스는 상대적으로 더 안전한 골목을 골라낸다"를 보여주고, "💯 만점 시나리오"로 전환해
   안전 스탬프가 거의 모두 채워지는 모습(⭐⭐⭐ + 다수 스탬프)을 보여주면 게이미피케이션 효과가
   극적으로 드러난다.
4. **Upstage 활용 강조**: 추천 경로 카드에 표시되는 문서 근거 배지를 클릭/포인트하며
   "이 문장은 실제 PDF 보고서를 Upstage Document Parse로 파싱하고 Information Extract로
   구조화 추출한 결과입니다"라고 설명.
5. **AI 설명 패널**: 부모용 리포트(시간 손실 대비 안전 이득)와 아이용 설명(쉬운 문장)을
   나란히 보여주며 "동일한 데이터인데 대상에 따라 다르게 설명합니다"를 강조.
   - 이때 아이용 카드 아래 뜨는 **"🎉 오늘의 안전 스탬프"** 보드(⭐⭐⭐ + 스탬프 칩)를
     함께 보여주며 "안전점수 81점이라는 숫자보다, 아이에게는 CCTV 지킴이·안전구역
     히어로 같은 눈에 보이는 보상으로 안전한 길을 스스로 선택하게 유도합니다"를 강조.
6. **차별화 클로징**: "범용 LLM에게 물었다면 이런 수치와 출처 없이 그럴듯한 말만 했을
   것입니다. 우리는 실제 공공데이터와 문서에 근거해 설명합니다."

## Before/After 비교 슬라이드용 문구

- **범용 LLM(예시)**: "이 길은 사람이 많이 다녀서 비교적 안전할 것으로 보입니다." (근거 없음)
- **우리 서비스**: "CCTV 12개(1km당 15개), 어린이보호구역 통과율 35%, 사고다발지역 0곳,
  'OO구 2024 통학로 안전진단보고서'에서 안전조치 완료로 확인된 구간입니다." (근거 있음)

## 예외 시나리오 (질의응답 대비)

- API 키가 없을 때: 화면 상단 배지가 `MOCK`으로 표시됨을 보여주며 "키만 넣으면 즉시
  실제 Tmap/공공데이터/Upstage로 전환됩니다"라고 설명.
- 임의의 다른 좌표를 입력해도 경로 3종(직선/큰길A/큰길B)이 항상 생성되고 점수가
  상대 정규화되어 계산됨을 보여줄 수 있음(단, 샘플 데이터는 데모 좌표 주변에만 있으므로
  다른 지역 좌표는 점수 차이가 작게 나올 수 있음 — 실제 데이터 연동 후 전국 커버).
